## Status Terms

- `inferred`: candidate behavior inferred from files or docs but not enough evidence exists.
- `needs_verification`: default for inferred or split-out features without focused evaluator evidence.
- `implemented_pending_evaluation`: implementation or docs exist, but relevant gates have not run after the latest change.
- `verified_pass`: relevant evidence and gates support the claim.
- `blocked`: required evidence cannot be collected because dependencies, sandbox, permissions, or environment failed.
- `deprecated`: intentionally retired with docs and compatibility notes.
