## Bugs Found and Fixed

### BUG-001: Refund demo evidence gaps ignored refund dispute type

- Area: GitHub Pages static demo.
- Symptom: The refund sample could miss the "damages calculation or refund allocation" evidence gap unless the same case also matched delivery or liability signals.
- Root cause: `docs/assets/app.js` checked `groups.includes("refund")`, but `refund` is not a keyword group. Refund appears under the `payment` keyword group and in the configured dispute type.
- Fix: Updated the evidence-gap rule to check `dispute.includes("refund")`.
- Files changed: `docs/assets/app.js`.
- Tests added/updated: `tests/test_docs_site.py::test_github_pages_app_is_static_and_wires_expected_actions` now guards against the nonexistent `groups.includes("refund")` check and requires the dispute-type refund check.
- Verification: `python -m pytest tests\test_docs_site.py` passed; full `python -m pytest` passed.

### BUG-002: README heading markers were not portable in local output

- Area: README rendering and internal navigation.
- Symptom: README emoji headings displayed as mojibake in the local PowerShell audit output, making the repository landing page harder to inspect and making anchor links depend on emoji-generated slugs.
- Root cause: Non-ASCII heading prefixes were present in a repository that otherwise normalizes text files and is audited from Windows shell tooling with non-UTF-8 console rendering.
- Fix: Replaced README emoji heading prefixes with ASCII headings and updated internal anchor links to match the new GitHub heading slugs.
- Files changed: `README.md`.
- Tests added/updated: `tests/test_docs_site.py::test_readme_internal_anchor_links_match_headings` verifies README hash links match actual headings.
- Verification: `python -m pytest tests\test_docs_site.py` passed; docs link checker passed for Markdown files.

### BUG-003: Static GitHub Pages surface had no regression coverage

- Area: Test suite / docs website stabilization.
- Symptom: The new static demo files were untested, so missing assets, broken form IDs, missing sample JSON, stale README image paths, accidental `docs/` ignore rules, or network-call regressions could pass the suite.
- Root cause: Existing tests covered Python package behavior but not the newly added public Pages entry point.
- Fix: Added a focused static docs test module.
- Files changed: `tests/test_docs_site.py`.
- Tests added/updated: New tests cover `docs/index.html`, static asset references, required form fields and buttons, app JS action wiring, absence of browser network calls, sample JSON validity, README local image references, README internal anchors, project identity, packaging metadata, and `.gitignore` docs hygiene.
- Verification: `python -m pytest tests\test_docs_site.py` passed; full `python -m pytest` passed.
