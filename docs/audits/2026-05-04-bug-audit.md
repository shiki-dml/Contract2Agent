## Fix Status

- AD-BUG-001: fixed. Auto patch targets now resolve inside the repo root, reject generated artifact directories including `.agentdoctor/`, `.git/`, `reports/`, and `traces/`, and use the strict default allowlist of root config files plus `prompts/*.md`.
- AD-BUG-002: fixed. `latest.json` baseline pointers must resolve inside `project_root/.agentdoctor/baselines`; escaping pointers are rejected with a warning instead of being read.
- AD-BUG-003: fixed. `triage --agent` paths that resolve outside `--project-root` are marked skipped, warned, and not read.
- AD-BUG-004: fixed. Baseline snapshot save skips an outside `agent_config_path` with a warning, without hashing, copying, or crashing.
- AD-BUG-005: fixed. Auto recommendations no longer include unsupported `--preview-patches`.
- AD-BUG-006: fixed. Patch preview now uses the strict prompt/config allowlist and treats root `prompt.md` as review-only by default.
- AD-BUG-007: fixed. Invalid deep rounds and auto target confidence now produce concise non-zero CLI errors without tracebacks, and auto validates before warnings.
- AD-BUG-008: fixed. Diagnostic Markdown includes a Time Cost section when measured time-cost data exists.
- AD-BUG-009: fixed. Baseline slowest-test timing uses `TestResult.duration_seconds`, falling back to round average only when per-test timing is unavailable.
- AD-BUG-010: fixed during second-pass verification. Patch preview reviewer notes no longer claim `--focus` or `--compare-baseline` are unsupported, and a regression test now covers the wording.

### Likely Bug Investigation Status

- AD-LIKELY-001: confirmed and fixed. Deep mode now enforces `MAX_DEEP_ROUNDS` and the CLI reports a concise error for larger values.
- AD-LIKELY-002: confirmed and fixed. Patch preview proposals are explicitly review-only by default and no longer advertise direct auto-apply eligibility.
- AD-LIKELY-003: confirmed and fixed. Auto mode now records post-patch validation confidence and lets successful post-patch validation drive final confidence/status when it reaches the target.
- AD-LIKELY-004: confirmed and fixed. Clean deep runs now use neutral recommendations and do not tell users to review failed/warning cases when none exist.

### Speculative Risk Investigation Status

- Speculative risk 1, path resolvers without containment: disproven as a remaining active read/write bug and documented as a guarded invariant. File inputs that must stay under a root use `resolve_within(...)` or are followed by `safe_read_text(...)`; new guard tests cover patch preview target escape and cost-estimate triage metadata paths outside the project root.
- Speculative risk 2, Typer versus argparse CLI behavior: confirmed and fixed. Typer accepted invalid choice-like values such as `deep --review bogus` and produced a traceback while argparse rejected them cleanly. Shared command handlers now validate choice-like options before work begins, and argparse now accepts the same `patch-preview --no-dry-run` switch exposed by Typer.
- Speculative risk 3, auto patching missing allowlisted files: documented as an intentional safety invariant, not a reproduced bug. Missing target creation is limited to strict root allowlisted config files, with a guard test covering the default `eval_config.yaml` target and ensuring no generated `.agentdoctor` path is created.

## 3. Architecture Map

Package and entry points:

- `pyproject.toml` defines distribution `agenttracedoctor`, import package `contract2agent`, and CLI scripts `c2a` and `agentdoctor` pointing at `contract2agent.cli:main`.
- `contract2agent/cli.py` owns all top-level CLI commands. It uses Typer if installed and falls back to argparse otherwise. In this environment it used argparse because `typer` is missing.

Core diagnostics:

- `contract2agent/diagnostic_modes.py` implements quick, deep, auto, diagnostic dataclasses, deterministic `SyntheticDiagnosticAgent`, `SafePatcher`, scoring, report generation, time-cost summary attachment, and Markdown/JSON report writers.
- `contract2agent/checker.py` implements deterministic trace validation against `AgentContract`.
- `contract2agent/schema.py` defines contract models and YAML loading.
- `contract2agent/parser.py` builds the built-in default paper-reader contract.

Failure taxonomy:

- `contract2agent/failure_taxonomy.py` defines `FailureType`, `Severity`, playbook entries, `FailureClassifier`, aggregation, fix strategy routing, taxonomy comparisons, and failure-type time-cost summaries.

Triage:

- `contract2agent/triage/discovery.py` discovers agent configs, prompts, tools, evals, baselines, and AgentDoctor config.
- `contract2agent/triage/parsers.py` reads/parses YAML/JSON/text and extracts agent/tool/eval/prompt signals.
- `contract2agent/triage/classifiers.py`, `risk.py`, `coverage.py`, `planner.py`, and `recommendations.py` classify agent type, assess risk, estimate coverage/cost, and choose next commands.
- `contract2agent/triage/report.py` writes triage Markdown/JSON.

Baselines and snapshots:

- `contract2agent/baseline.py` implements baseline save/load/compare, agent state snapshots, file hashing/copying, git metadata capture, comparison summaries, rollback recommendations, and baseline Markdown/JSON writers.

Patch preview and patch safety:

- `contract2agent/patch_preview/cli.py` orchestrates preview generation.
- `contract2agent/patch_preview/loader.py` loads findings from diagnostic reports.
- `contract2agent/patch_preview/target_selection.py` enforces patch target allowlists.
- `contract2agent/patch_preview/security.py` excludes secret-like paths and redacts secret-like text.
- `contract2agent/patch_preview/strategies.py`, `risk.py`, `diff_builder.py`, `validation.py`, `baseline.py`, and `report.py` select fix strategies, compute risk/approval flags, build diffs, create validation plans, attach baseline impact, and write reports.

Time/cost:

- Measured diagnostic timing is attached in `diagnostic_modes.py` via `TestResult.duration_seconds`, round timestamps, `DiagnosticRound.time_cost_summary`, and `DiagnosticReport.time_cost_summary`.
- Static pre-run cost estimation lives in `contract2agent/cost_estimate/` and is exposed by `agentdoctor cost-estimate`.
- Baseline time summaries are derived in `baseline.py`.

Tests and docs:

- Tests cover triage, diagnostic modes, patch preview, failure taxonomy, baselines, cost estimate, and legacy contract-to-agent behavior under `tests/`.
- Documentation is under `README.md`, `docs/`, `examples/`, and `mkdocs.yml`.
- CI currently includes `.github/workflows/docs.yml` for docs build/deploy only.

## 4. Confirmed Bugs

### AD-BUG-001

Severity: critical

Confidence: confirmed

Component: auto mode patch safety

Files/functions involved:

- `contract2agent/diagnostic_modes.py:is_safe_patch_target` lines 1418-1443
- `contract2agent/diagnostic_modes.py:SafePatcher.apply` lines 519-547

Exact evidence:

- `is_safe_patch_target` rejects `reports`, `traces`, `.git`, and `__pycache__`, but does not reject `.agentdoctor/baselines`.
- It returns `True` for any exact safe filename such as `eval_config.yaml` anywhere under the repo.
- Reproduction showed a baseline artifact was considered safe and overwritten.

Reproduction command:

```powershell
python -c "from pathlib import Path; from contract2agent.diagnostic_modes import SafePatcher, PatchProposal, is_safe_patch_target; root=Path('<temporary-audit-dir>/auto_baseline_patch'); target=root/'.agentdoctor/baselines/baseline_1/eval_config.yaml'; print(is_safe_patch_target(target, root)); patch=SafePatcher(root).apply(PatchProposal(file_path=target,new_text='baseline: changed\n',patch_summary='test',reason_for_patch='test'), round_index=1, previous_confidence=0.1); print(target.read_text(encoding='utf-8').strip()); print(patch.files_changed)"
```

Observed output:

```text
True
baseline: changed
['.tmp\\bug_audit\\auto_baseline_patch\\.agentdoctor\\baselines\\baseline_1\\eval_config.yaml']
```

Expected behavior:

Auto mode must never patch baseline files or generated AgentDoctor artifacts.

Actual behavior:

A file under `.agentdoctor/baselines/.../eval_config.yaml` is accepted as safe and overwritten.

Root cause:

The safety check is filename-based and only denies a small set of directories.

User impact:

Auto repair can corrupt saved baselines or generated artifacts if they contain an allowlisted filename.

Suggested fix:

Reject `.agentdoctor/baselines`, `.agentdoctor/patches`, `.agentdoctor/triage`, `.agentdoctor/cost`, generated report trees, and any path outside a strict allowlist rooted at exactly the expected locations.

Regression test to add:

`test_auto_patcher_rejects_agentdoctor_baseline_paths`.

Safe to auto-fix:

Yes, with focused tests.

### AD-BUG-002

Severity: critical

Confidence: confirmed

Component: baseline loading / path traversal

Files/functions involved:

- `contract2agent/baseline.py:load_baseline` lines 1401-1422
- `contract2agent/baseline.py:_resolve_under_root` lines 2410-2415

Exact evidence:

- `load_baseline(..., "latest")` reads `.agentdoctor/baselines/latest.json`.
- If it contains `path`, the code resolves that path but does not verify it remains under the project root.
- Reproduction loaded a baseline JSON outside the declared project root.

Reproduction command:

```powershell
python -c "from contract2agent.baseline import load_baseline; r=load_baseline('<temporary-audit-dir>/baseline_escape/project','latest'); print(r.baseline_path); print(r.baseline.get('baseline_id') if r.baseline else None); print(r.warnings)"
```

Observed output:

```text
<temporary-audit-dir>\baseline_escape\outside_baseline.json
outside_project_baseline
[]
```

Expected behavior:

A latest-baseline pointer must be rejected if it escapes the project root or baselines directory.

Actual behavior:

The loader follows `../outside_baseline.json` and reads it without warning.

Root cause:

`_resolve_under_root` resolves but does not enforce containment.

User impact:

A malicious or corrupted `latest.json` can make baseline compare read arbitrary JSON files outside the project. If those files contain sensitive data, reports may leak it.

Suggested fix:

Replace `_resolve_under_root` usages for baseline refs with a resolver that raises or returns a warning unless the resolved path is under `root/.agentdoctor/baselines`.

Regression test to add:

`test_load_baseline_latest_pointer_cannot_escape_project_root`.

Safe to auto-fix:

Yes, with compatibility handling for old relative baseline refs.

### AD-BUG-004

Severity: high

Confidence: confirmed

Component: baseline snapshot save

Files/functions involved:

- `contract2agent/baseline.py:discover_snapshot_files` lines 1442-1463
- `contract2agent/baseline.py:copy_snapshot_file` lines 1517-1534
- `contract2agent/baseline.py:_relative` lines 2417-2425

Exact evidence:

Saving a baseline with an agent config path outside the project crashes with `shutil.SameFileError`.

Reproduction command:

```powershell
python -c "from pathlib import Path; import json; from contract2agent.diagnostic_modes import run_quick_diagnosis; from contract2agent.baseline import save_baseline; root=Path('<temporary-audit-dir>/snapshot_escape/project'); root.mkdir(parents=True, exist_ok=True); report=run_quick_diagnosis(out_dir=root/'reports'); result=save_baseline(report=report, project_root=root, agent_config_path='../outside_agent.yaml')"
```

Observed failure:

```text
shutil.SameFileError: WindowsPath('<temporary-audit-dir>/snapshot_escape/outside_agent.yaml') and WindowsPath('<temporary-audit-dir>/snapshot_escape/outside_agent.yaml') are the same file
```

Expected behavior:

The outside path should be rejected with a warning or clean error before hashing/copying.

Actual behavior:

Snapshot construction tries to treat the outside path as safe and crashes.

Root cause:
